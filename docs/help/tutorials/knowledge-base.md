# 知识库系统教程

知识库让 Memfit 能基于你的资料工作，而不是只依赖模型内置知识。它适合保存产品文档、团队规范、漏洞资料、项目说明、历史经验和任何需要长期复用的领域材料。

【配图：知识库页面总览，标注知识库列表、创建入口、导入入口、诊断入口】

## 知识库适合放什么

| 内容类型 | 示例 | 适用场景 |
| --- | --- | --- |
| 产品文档 | API 文档、部署文档、用户手册 | 让 AI 基于产品事实回答 |
| 安全资料 | CVE、漏洞复现、加固基线 | 安全分析、审计、报告生成 |
| 团队规范 | 编码规范、测试规范、交付标准 | 代码审查、文档检查 |
| 项目资料 | 架构说明、模块设计、变更记录 | 项目理解、交接、排错 |
| 经验总结 | 常见问题、处理流程、排障记录 | 复用团队经验 |

如果一段资料会被多次引用，就适合放进知识库。

## 创建知识库

在知识库页面点击 **新建知识库**，填写：

| 字段 | 建议 |
| --- | --- |
| **知识库名称** | 使用清晰名称，例如“Web 安全基线”“项目 A 运维手册”。 |
| **描述** | 写明资料来源、适用范围和更新时间。 |
| **类型 / 标签** | 用于后续筛选和管理。 |
| **知识库长度** | 控制单条知识片段长度；不确定时保留默认值。 |

【配图：新建知识库弹窗，标注名称、描述、类型、长度】

## 添加资料

Memfit 支持多种添加方式。

### 拖拽文件

在 Agent 首页或知识库区域拖拽文件，Memfit 会把文件加入默认或指定知识库。适合快速把一批文档交给 AI。

【配图：拖拽文件创建或更新知识库】

### 手动导入文档

在知识库详情中选择导入文件。建议优先导入结构清晰的 `.md`、`.txt`、文档导出文本或规范化资料。

### 手动添加知识条目

当你只想添加一条经验或规则时，可以直接新增知识条目。建议写成“场景 + 结论 + 证据/来源”的形式。

```text
场景：Node.js 项目依赖审计
结论：升级依赖前需要先运行测试和构建，避免锁文件变化导致部署失败。
来源：团队发布流程 2026 版
```

## 构建知识库

资料加入后，需要构建知识库。构建过程会完成分块、索引和可检索数据生成。

常见构建参数：

| 参数 | 作用 |
| --- | --- |
| **chunk** | 控制分块策略。文档结构清晰时可用默认；长文档可适当调大。 |
| **entrylen** | 控制知识片段长度。太短会丢上下文，太长会影响检索精度。 |
| **concurrency** | 构建并发数。机器性能一般时不要过高。 |
| **disableERM** | 是否关闭实体关系模型能力。需要实体和关系分析时保持默认。 |

【配图：构建知识库参数面板】

## 诊断知识库可用性

构建完成后，建议使用 **知识库可用性诊断** 检查：

1. 知识库是否能被正常检索。
2. 查询是否返回相关内容。
3. 知识片段是否过长或过短。
4. 模型是否能基于检索结果回答。

可以用一个最能代表资料的问题做测试：

```text
根据这份知识库，系统上线前必须检查哪些安全项？
```

## 在 Agent 中使用知识库

知识库创建并构建后，可以在 Agent 对话中使用。

### 从资源区选择

1. 打开 Agent 主界面的知识库资源区。
2. 找到目标知识库。
3. 选择或挂载到当前对话。
4. 输入任务，让 AI 明确“基于知识库回答”。

```text
请基于我挂载的“Web 安全基线”知识库，检查这份 Nginx 配置是否符合要求。
```

【配图：在 Agent 资源区选择知识库】

### 在输入框中引用

如果界面支持 Mention，可以在输入框中选择知识库，让当前任务显式绑定该知识库。

【配图：输入框 Mention 知识库】

## 管理知识库

知识库不是一次性资料仓库，需要定期维护：

| 操作 | 什么时候做 |
| --- | --- |
| **更新资料** | 文档版本变化、规则更新、项目变更后。 |
| **重新构建** | 新增或删除大量资料后。 |
| **清空知识库** | 资料来源错误或需要完全重建时。 |
| **查询知识条目** | 检查某条资料是否已经进入索引。 |
| **查看实体 / 向量** | 排查检索质量、分析知识结构时。 |

## 写好知识库资料的建议

- 每份资料有明确标题。
- 长文档使用层级标题分段。
- 避免把完全无关的资料放在同一个知识库。
- 对规范类内容写明适用版本。
- 对安全资料保留来源和时间。
- 构建后用真实问题做一次诊断。

## 常见问题

### AI 没有引用知识库内容

确认知识库已构建，并且当前对话已挂载知识库。提问时明确说明“基于这个知识库回答”。

### 检索结果不准确

检查资料是否过杂、标题是否清晰、分块是否过长。必要时拆成多个主题知识库。

### 导入后没有生效

新增资料后需要重新构建或等待构建完成。构建失败时先运行可用性诊断。
