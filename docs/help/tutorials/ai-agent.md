# Agent 使用教程

AI Agent 是 Memfit 的主工作区。它不只是聊天窗口，而是一个可以读取上下文、调用工具、进入专注模式、请求 Review 并执行任务的工作台。

本章按日常使用顺序介绍：先设置权限，再输入任务和 `@` 上下文，最后学习如何指定工具、知识库和专注模式。

【配图：AI Agent 主工作区，标注输入框、@ 上下文入口、Review 面板、资源面板、专注模式入口、模型/用量入口】

## 先理解权限控制

Agent 能做多少事，取决于你给它的权限。第一次使用前，建议先完成 [基础配置与自定义 AI 配置](/docs/help/quick-start/tier-ai) 中的权限控制。

常见配置含义：

| 配置 | 含义 | 建议 |
| --- | --- | --- |
| **激活系统文件操作权限** | 允许 AI 读取、生成或修改你授权范围内的文件。 | 真实项目分析通常需要开启；只想聊天可以关闭。 |
| **禁用 Tools** | 开启后 AI 不能调用工具，只能对话。 | 想让 Agent 执行任务时保持关闭。 |
| **Review 规则：Manual** | 工具调用、计划、文件操作等需要你手动确认。 | 新手和真实项目推荐。 |
| **Review 规则：AI** | 由 AI 根据风险判断是否需要继续。 | 适合低风险、重复性任务。 |
| **Review 规则：Yolo** | 尽量自动执行，减少确认。 | 只建议在沙箱、只读、低风险任务中使用。 |
| **禁用工具运行时 AI 审查** | 工具运行中跳过 AI 审查。 | 会提升自动化，但也减少风险拦截。 |
| **风险阈值** | 决定哪些动作可自动通过，哪些动作需要确认。 | 阈值越宽松，自动执行越多。 |

【配图：基础配置面板，标注文件权限、Review 规则、风险阈值、禁用 Tools、禁用工具运行时 AI 审查】

### 如何设置全自动执行

如果你明确要让 Agent 尽量自动跑完整个任务，可以这样设置：

1. 开启 **激活系统文件操作权限**。
2. 保持 **禁用 Tools** 关闭。
3. 将 **Review 规则** 设置为 `Yolo`。
4. 根据任务风险适当提高 **风险阈值**。
5. 如需进一步减少中途停顿，再开启 **禁用工具运行时 AI 审查**。

:::warning
全自动执行会降低人工确认和风险拦截。它可能读取错误目录、调用不合适的工具、生成或修改非预期文件，也可能在网络请求、命令执行、批量文件操作中放大错误。建议只在沙箱环境、只读任务、演示任务或你完全理解影响范围时使用。
:::

### 手动执行和 AI 审核有什么意义

`Manual` 不是为了拖慢任务，而是把关键决策权留给你。它适合这些场景：

- 文件修改、删除、覆盖、批量生成。
- 网络访问、扫描、请求外部目标。
- 命令执行、脚本运行、工具链调用。
- 涉及私密文件、客户资料、凭据或生产环境。

`AI` 审核则适合在效率和安全之间折中。AI 会根据工具名、参数、上下文和风险提示判断是否继续，但它不能替代你对真实业务边界的判断。

## 输入任务开始执行

在底部输入框用自然语言描述任务即可。你不需要写固定命令，Agent 会根据任务内容判断是否直接回答、生成计划、调用工具或请求 Review。

一个好的任务通常包含目标、范围、输出格式和权限边界：

```text
请只读分析当前项目，说明主要目录、入口文件、配置方式和启动命令。
输出一份面向新开发者的 Markdown 说明，不要修改任何文件。
```

【配图：输入框中填写自然语言任务，标注目标、范围、输出格式、权限边界】

如果任务复杂，可以先要求它出计划：

```text
先不要执行工具。请先给出你准备如何分析这个项目的计划，等我确认后再继续。
```

## 使用 @ 引入更多上下文

输入任务时可以使用 `@` 系统把文件、知识库、工具、SKILL 或专注模式带入当前任务。`@` 的意义是把“你希望 AI 重点参考或使用的对象”显式告诉 Agent。

常见用法：

| @ 对象 | 适合场景 | 示例 |
| --- | --- | --- |
| **@文件 / @目录** | 让 AI 分析指定文件、日志、配置或项目目录。 | “请基于 @README.md 和 @package.json 分析启动方式。” |
| **@知识库** | 让 AI 按团队规范、产品文档、漏洞库回答。 | “请基于 @团队安全规范 审计这段鉴权逻辑。” |
| **@工具** | 指定 AI 使用某个工具完成检查、查询或转换。 | “请使用 @TLS证书检查工具 检查 example.com。” |
| **@SKILL** | 使用一套固定场景流程。 | “请用 @Java代码审计Skill 分析 @src/main/java/auth。” |
| **@专注模式** | 让 Agent 进入更适合某类任务的执行循环。 | “请用 @代码安全审计专注模式 分析这个模块。” |

【配图：输入框触发 @ 菜单，展示文件、知识库、工具、SKILL、专注模式候选项】

你也可以同时使用自然语言和多个 `@`：

```text
请基于 @团队安全编码规范，使用 @代码搜索工具，
只读分析 @src/auth 目录的鉴权绕过风险。
输出风险等级、证据路径、触发条件和修复建议。
```

## AI 可以看到你的用户空间

当你开启系统文件操作权限后，Agent 可以在你授权的用户空间中读取任务相关文件，并在 Review 允许后执行文件操作。这意味着你可以让它处理真实项目，而不需要把所有内容复制进聊天框。

常见操作：

- 让 AI 阅读当前项目目录并总结结构。
- 引入某个配置文件，让 AI 判断配置问题。
- 引入日志文件，让 AI 分析异常原因。
- 引入代码目录，让 AI 做只读审计或生成报告。
- 允许 AI 生成文档、脚本或修复建议文件。

【配图：选择本地文件或目录作为上下文，标注用户空间、项目目录、只读/写入 Review】

使用真实文件时建议：

1. 先给出明确范围，例如“只看 `src/auth`，不要进入 `node_modules`”。
2. 涉及写入时保留 `Manual Review`。
3. 不要无意引入密钥、客户数据、个人隐私或生产凭据。
4. 要求 AI 在结论中给出文件路径和证据位置，方便复核。

## 指定 AI 使用特定工具或知识库

如果你希望 Agent 不要自由选择，而是使用指定工具或知识库，可以通过 `@` 明确指定，并在任务中写出使用边界。

指定知识库：

```text
请只基于 @Memfit使用手册 回答：
如何配置基础 AI 模型和权限控制？
如果知识库里没有答案，请明确说“资料中未提到”。
```

指定工具：

```text
请使用 @TLS证书检查工具 检查 example.com 的证书有效期。
不要使用其它网络扫描工具。
```

指定知识库和工具组合：

```text
请基于 @团队安全规范，使用 @代码搜索工具，
检查 @src/auth 是否存在未鉴权访问。
只读分析，不要修改文件。
```

【配图：@ 知识库和 @ 工具同时出现在输入框中，标注“指定使用”和“限制其它工具”】

如果 Agent 仍然选择了不合适的工具，可以在 Review 中拒绝，并补充一句：

```text
拒绝这次工具调用。请只使用我 @ 的工具，不要调用其它工具。
```

## 处理计划和 Review

复杂任务开始时，Memfit 可能会生成计划或任务树。你需要检查：

1. 目标是否理解正确。
2. 范围是否过大。
3. 是否会访问敏感路径。
4. 是否会修改文件或调用高风险工具。
5. 是否使用了你指定的知识库、工具、SKILL 或专注模式。

当 Agent 准备执行工具、文件操作或 SKILL 时，Memfit 会根据你的配置触发 Review。

| Review 类型 | 你会看到什么 | 建议 |
| --- | --- | --- |
| **计划 Review** | 任务步骤、目标和执行方式 | 确认方向正确再继续。 |
| **工具 Review** | 工具名称、参数、可能影响 | 检查目标、路径、参数和风险。 |
| **文件 Review** | 读取、写入、修改或生成的文件 | 写入类操作建议手动确认。 |
| **SKILL Review** | SKILL 名称、描述、参数 | 确认场景和输入无误。 |
| **专注模式 Review** | 即将进入的专注模式和目标 | 确认模式适合当前任务。 |

【配图：Review 面板，标注工具名、参数、风险说明、允许/拒绝/补充要求】

## 专注模式是什么

专注模式是为某一类任务定制的 Agent 执行循环。普通 Agent 会根据任务自由规划；专注模式会提前配置好更适合该场景的提示词、工具、动作、执行步骤和收尾方式。

你可以把它理解为“带专业工作流的 Agent”。

专注模式的意义：

- 限定 Agent 的行动空间，减少跑偏。
- 为复杂任务提供更稳定的步骤。
- 自动加载适合该场景的工具和输出格式。
- 让用户不用每次都重复写长提示词。
- 在安全审计、流量分析、报告生成等场景中提高一致性。

【配图：专注模式选择入口，标注通用模式、代码安全审计、知识增强问答、报告生成等模式】

## 典型好用的专注模式

实际可用名称会随版本变化，以下是常见类型和适用场景：

| 专注模式 | 适合任务 | 什么时候用 |
| --- | --- | --- |
| **通用 ReAct / 默认模式** | 普通问答、轻量分析、自由探索。 | 任务不属于明确领域时。 |
| **智能问答 / 知识增强问答** | 基于知识库回答问题。 | 需要严格参考文档、规范或手册时。 |
| **目录探索** | 理解项目结构、文件入口、模块关系。 | 第一次接触一个项目时。 |
| **报告生成** | 汇总分析结果并生成报告。 | 已有材料，需要结构化输出时。 |
| **代码安全审计** | 分析鉴权、注入、路径穿越、敏感信息等风险。 | 对代码目录做安全检查时。 |
| **HTTP 流量分析** | 分析请求、响应、接口行为和异常流量。 | 有 HTTP 流量或接口记录时。 |
| **信息安全侦察** | 多工具组合的信息收集。 | 明确授权的安全测试和资产分析中。 |
| **SyntaxFlow 规则编写** | 生成或调试 SyntaxFlow 规则。 | 需要把审计思路沉淀成规则时。 |
| **Yaklang / Python 脚本生成** | 生成脚本辅助处理任务。 | 需要自动化脚本但希望 Agent 帮你起草时。 |

使用示例：

```text
请进入 @代码安全审计专注模式，
只读分析 @src/auth 的鉴权绕过风险。
先给出计划，等我确认后再执行。
```

如果你想了解专注模式的开发和扩展，请阅读顶部导航中的 **开发者指南**。

## 观察执行并验收结果

任务执行时，对话流会展示 Agent 的关键过程：计划、行动、观察、阶段性总结和最终回复。

验收时重点看：

1. 是否遵守你指定的范围和权限边界。
2. 是否使用了你 `@` 的文件、知识库、工具或专注模式。
3. 是否给出证据路径、工具结果或知识库来源。
4. 是否输出了可执行结论，而不是泛泛描述。

可以继续追问：

```text
把结论整理成一份给研发团队看的修复清单。
每条都保留证据路径和优先级。
```

## 常见任务模板

### 项目理解

```text
请使用 @目录探索专注模式，只读分析 @当前项目。
说明目录结构、主要入口、配置方式和启动命令。
不要修改文件。
```

### 代码审计

```text
请使用 @代码安全审计专注模式，基于 @团队安全规范，
只读审计 @src/auth。
重点关注身份认证、权限校验和敏感信息泄露。
输出风险等级、证据位置和修复建议。
```

### 知识库问答

```text
请只基于 @Memfit使用手册 回答：
如何配置权限控制和自定义 AI 模型？
不要使用知识库之外的信息。
```

### 指定工具执行

```text
请使用 @TLS证书检查工具 检查 example.com。
只输出证书主体、颁发者、有效期、剩余天数和风险提示。
```
