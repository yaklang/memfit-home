# 基础配置与自定义 AI 配置

完成安装并启动 Memfit 后，建议先完成两类配置：

1. **基础配置**：决定 AI Agent 是否可以使用工具、是否需要人工确认、是否允许访问本地文件、任务最多运行多少轮。
2. **（可选）自定义 AI 模型**：添加自己的 AI 厂商、模型、API Key、BaseURL / Endpoint，并设置高质模型、轻量模型、视觉模型的调用策略。

这两类配置共同决定 Memfit 能否稳定、安全地完成任务。第一次使用时，可以先按本文的“推荐入门配置”完成最小可用配置，再根据实际任务调整。

![AI 模型添加表单基础项](/img/help/quick-start/ai-model-add-basic.png)

## 进入配置页面

在 Memfit 主界面中打开配置入口：

1. 点击界面左侧边栏的 **AI 模型**。
2. 在模型列表顶部点击 **配置**、**添加** 或 **刷新**。配置按钮用于进入全局运行参数；添加按钮用于新增第三方模型。
3. 在对话配置面板中调整基础配置。
4. 进入 **AI 模型配置** 区域，添加或选择 AI 模型。

如果你只是想尽快开始使用，确认 **使用默认系统配置AI** 已开启，然后检查权限控制即可。如果你需要使用自己的 API Key 或私有模型服务，再继续完成自定义 AI 配置。

![从 AI 模型列表进入添加模型](/img/help/quick-start/ai-model-add-entry.png)

## 基础配置

基础配置来自 Memfit 的 AI Agent 运行参数，主要控制“AI 能做什么、什么时候需要用户确认、任务失败时如何重试”。

### 核心权限

![Review 规则与核心权限配置](/img/help/quick-start/review-rule-settings.png)

核心权限用于设置人在 AI 系统中的参与度。图中最重要的是 **Review 规则**：`Manual` 表示关键动作交给用户确认，`AI` 表示由 AI 辅助判断是否放行，`Yolo` 表示尽量自动执行、减少打断。输入框下方的小图标会同步反映当前审查和自动化策略，便于你确认当前运行模式。

:::tip

开启 YOLO 模式虽然会减少打断，但人对系统的控制会减弱。涉及文件修改、系统命令、扫描或生产环境操作时，建议先使用 `Manual`，确认流程可靠后再提高自动化程度。

:::

### 权限控制

| 配置项 | 建议 | 作用 |
| --- | --- | --- |
| **激活系统文件操作权限** | 按需开启 | 允许 AI 使用系统文件操作能力，例如读取任务相关文件、生成或修改文件。处理真实项目文件时通常需要开启；只想聊天或演示时可以关闭。 |
| **禁用Tools** | 默认关闭 | 开启后，AI 不再调用外部工具，Memfit 会退化为纯对话模式。 |
| **禁用工具运行时AI审查** | 默认关闭 | 开启后，工具运行时跳过 AI 审查，可提升执行效率，但会降低自动检查风险的能力。 |
| **Review 规则** | 新手选 Manual | 控制高风险动作如何审批：`Manual` 需要人工确认，`AI` 由 AI 判断，`Yolo` 更偏自动执行。 |
| **风险阈值** | 默认 `0.5` | 当风险评分低于阈值时可自动同意；高于阈值时转为手动确认。 |

:::tip
第一次使用建议保留 **Manual** 审查策略，并开启 **激活系统文件操作权限**。这样 AI 可以处理本地文件，但关键动作仍会停下来让你确认。
:::

![权限配置面板](/img/help/quick-start/permission-settings-panel.png)

### 人机交互控制

| 配置项 | 建议 | 作用 |
| --- | --- | --- |
| **禁用人机交互** | 默认开启 | 开启后，AI 不会频繁向用户追问问题，适合让任务连续执行。 |
| **允许任务规划阶段人机交互** | 默认开启 | 允许 AI 在制定任务计划时向用户补充确认需求。 |
| **任务规划阶段人机交互次数** | 默认 `3` | 限制规划阶段最多追问几次。 |
| **用户交互测试** | 按需设置 | 限制整个任务中 AI 主动询问用户的最大次数。 |

如果任务目标明确，例如“分析这个目录并生成报告”，可以开启 **禁用人机交互**，让 AI 尽量直接完成。如果任务还不清楚，保留规划阶段交互，AI 会先把问题问清楚。

### 任务执行与上下文限制

| 配置项 | 默认值 | 作用 |
| --- | --- | --- |
| **AI对话重试次数** | `3` | 远端 AI 调用不稳定时，单次对话最多重试次数。 |
| **AI事务重试次数** | `5` | 回答质量不高或执行失败时，任务事务级别的重试次数。 |
| **ReAct 迭代轮数限制** | `100` | 限制 ReAct 执行循环最多运行多少轮，避免任务无限执行。 |
| **时间线上下文限制** | `100` | 限制时间线中保留的上下文条目数量。 |
| **时间线上下文大小** | `60 K` | 限制时间线上下文内容大小。 |
| **压力token阈值** | `40 K` | 当对话 token 压力超过阈值时提醒或触发压缩策略。 |

日常使用建议保留默认值。只有在任务经常中途停止时，再适当提高 **ReAct 迭代轮数限制**；如果上下文过长导致模型响应慢，可以降低时间线相关限制。

## 自定义 AI 配置

Memfit 支持按模型用途管理多个 AI 配置。实现上会把模型分为三类：

| 模型类型 | 用途 | 适合配置 |
| --- | --- | --- |
| **高质模型** | 复杂规划、深度推理、代码分析、安全分析 | 推理质量更强的模型 |
| **轻量模型** | 高频、简单、低成本的分类、摘要、路由任务 | 响应快、成本低的模型 |
| **视觉模型** | 截图、界面、图表、图片内容理解 | 支持图像输入的模型 |

### 添加 AI 模型

在 **AI 模型配置** 中点击 **添加**，按表单填写：

![添加第三方模型基础表单](/img/help/quick-start/third-party-model-basic-form.png)

| 表单项 | 如何填写 |
| --- | --- |
| **厂商** | 选择内置厂商，或填写自定义兼容 OpenAI 风格的厂商名称。 |
| **模型类型** | 选择 **高质模型**、**轻量模型** 或 **视觉模型**。 |
| **APItype** | 选择模型接口风格，常见值为 `chat_completions` 或 `responses`。 |
| **模型名称** | 选择或输入模型名，例如 `gpt-4o`、`gpt-4o-mini`、`deepseek-chat`。 |
| **ApiKey** | 填写对应厂商的 API Key / Token。部分内置免费服务可能不需要真实 Key。 |

填写完成后，可以先点击 **测试并添加**。如果测试成功，再点击 **确定添加** 保存配置；如果模型名称无法自动获取，可以点击“点击刷新”重新拉取可用模型列表。

### 配置自定义 BaseURL 或 Endpoint

如果使用私有部署、代理服务或兼容 OpenAI 的第三方网关，展开 **高级配置**：

![添加第三方模型高级配置](/img/help/quick-start/third-party-model-advanced-form.png)

| 高级配置 | 适用场景 |
| --- | --- |
| **BaseURL** | 服务保持 OpenAI 风格路径时使用，例如 `https://api.openai.com/` 或 `https://api.example.com/v1`。 |
| **启用Endpoint** | 服务不是标准 OpenAI 路径时开启。 |
| **Endpoint** | 开启 Endpoint 后填写完整请求地址，例如 `https://api.example.com/v1/chat/completions`。 |
| **代理地址** | 网络需要代理时填写，例如 `http://127.0.0.1:7890`。 |
| **Header** | 需要额外请求头时添加，例如企业网关要求的租户标识。 |
| **Enable Thinking** | 支持显式推理开关的模型可按需配置；不支持时保持为空。 |
| **Max Tokens / Temperature / Top-P / Top-K** | 用于控制输出长度、随机性和采样范围；不确定时保持为空，使用模型默认值。 |
| **Frequency Penalty** | 用于降低重复表达；日常任务通常不需要设置。 |
| **Reasoning Effort** | 支持推理强度的模型可设置低、中、高等档位；复杂分析任务可适当提高。 |

:::note
`BaseURL` 与 `Endpoint` 通常二选一。标准 OpenAI 兼容服务优先填 `BaseURL`；非标准接口再开启 `Endpoint` 并填写完整地址。
:::

### 选择当前使用的模型

添加模型后，回到模型列表：

1. 展开 **高质模型**、**轻量模型** 或 **视觉模型** 分组。
2. 找到刚添加的模型。
3. 点击选择或设为当前模型。
4. 至少配置一个 **高质模型**。如果希望降低成本，再配置一个 **轻量模型**。

Memfit 在任务运行时会记录不同模型层级的 token 消耗、首次响应耗时和总耗时，方便你后续判断是否需要调整模型组合。

### 设置调用模式

在 AI 模型配置底部设置 **调用模式**：

| 调用模式 | 说明 | 推荐场景 |
| --- | --- | --- |
| **Auto** | 根据任务内容自动选择模型 | 推荐默认使用 |
| **Performance** | 优先使用高质模型 | 复杂推理、质量优先 |
| **Cost** | 优先使用轻量模型 | 高频任务、预算敏感 |
| **Balance** | 在质量、速度、成本之间平衡 | 日常通用 |

如果你只配置了一个模型，建议选择 **Auto**。如果同时配置了高质模型和轻量模型，日常仍建议使用 **Auto** 或 **Balance**。

### 设置降级策略

**禁用降级** 控制模型不可用时是否允许自动切换：

- 保持关闭：高质模型不可用时，Memfit 可以尝试使用轻量模型继续执行，适合大多数场景。
- 开启禁用降级：模型不可用时直接失败，适合必须固定模型、需要严格审计的场景。

## 推荐入门配置

第一次使用可以这样配置：

1. 基础配置中开启 **激活系统文件操作权限**。
2. **Review 规则** 选择 `Manual`。
3. 保持 **禁用Tools** 关闭。
4. 保持 **使用默认系统配置AI** 开启；如果你要使用自己的 Key，则添加一个高质模型。
5. 高质模型配置为你最可靠的主力模型。
6. 轻量模型可选，用于降低高频任务成本。
7. 调用模式选择 **Auto**。
8. 保持 **禁用降级** 关闭。

完成后，回到主界面输入一个简单任务，例如：

```text
请总结当前项目目录的结构，并指出主要入口文件。
```

如果 AI 能生成计划、请求必要权限并给出结果，说明基础配置与 AI 配置已经可用。

## 常见问题

### AI 无法读取或修改本地文件

检查 **激活系统文件操作权限** 是否开启，并确认 Review 弹窗中的文件操作已被允许。如果你开启了 **禁用Tools**，AI 将无法调用外部工具。

### API Key 已填写但模型不可用

检查 **厂商**、**APItype**、**模型名称** 是否匹配。使用第三方兼容服务时，再检查 **BaseURL** 或 **Endpoint** 是否填写正确。

### AI 一直要求确认

检查 **Review 规则** 和 **风险阈值**。新手建议保持 Manual；熟悉任务后，可把 Review 规则改为 AI，或适当提高风险阈值。

### 任务运行太久

降低 **ReAct 迭代轮数限制**，或缩小任务范围。如果任务需要处理大量文件，优先让 AI 先列计划，再分批执行。
